### Bulk Assign Bug Fix Prompt

We have a critical issue in the CRM Lead Management system.

#### Problem

When multiple leads are selected and **Bulk Assign** is clicked, the system displays **"Assign Failed"** and no leads are assigned to the selected user.

#### Required Investigation & Fix

##### Frontend

* Verify selected lead IDs are collected correctly.
* Verify the selected employee/user ID is included in the request.
* Check that the API request payload is properly formatted.
* Display actual API error messages instead of generic "Assign Failed".
* Add loading and success states.

##### Backend

* Verify Bulk Assign API endpoint is working correctly.
* Validate request payload structure.
* Ensure lead IDs are received as an array.
* Ensure assigned user ID exists and is valid.
* Verify authorization and permissions.
* Check for validation failures and database errors.
* Add detailed logging for request, response, and exceptions.

##### Database

* Verify lead records exist.
* Verify assigned_to/user_id field is present and updateable.
* Check foreign key constraints.
* Ensure bulk update query updates all selected leads.
* Prevent partial updates by using database transactions.

##### API Validation

Expected payload example:

```json
{
  "leadIds": [1,2,3,4,5],
  "assignedTo": 7
}
```

Expected response:

```json
{
  "success": true,
  "message": "5 leads assigned successfully"
}
```

##### Testing

* Test assigning 1 lead.
* Test assigning 10 leads.
* Test assigning 100+ leads.
* Test assigning leads across multiple pagination pages.
* Verify UI updates immediately after assignment.
* Verify database records are updated correctly.

##### Additional Checks

* Verify Bulk Assign works with pagination.
* Verify selected leads from different pages are processed correctly.
* Verify no duplicate API calls are generated.
* Verify CSRF/Auth token is included in request.
* Verify session expiration is not causing assignment failures.

#### Expected Result

* Bulk Assign successfully assigns all selected leads to the chosen user.
* Success message shows the number of leads assigned.
* UI refreshes automatically and reflects new ownership.
* No "Assign Failed" message appears unless a genuine error occurs.

**Important:** Do not implement a workaround. Identify and fix the root cause in frontend, backend, API, authentication, permissions, and database layers. Add proper error handling, logging, and production-ready validation.
